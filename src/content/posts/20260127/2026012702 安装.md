## 一.前言

> 安装软件时，一些特殊格式的应用无法直接安装、安装的时候会联网扫描安装包、检测到黑名单应用无法安装、无法降级安装等等，情况就是这么个情况。

> 这些限制的出发点是好的，应用安全性确实拉满，对于不太懂手机的人来说的确能多一份保护，不过对于喜欢折腾的人来说也少了点自由。

## 二.InstallerX_Revived - 开源的第三方安装器

1. InstallerX-Revived。这是一个开源的第三方安装器，基于 InstallerX（已停更） 二次开发，功能更加全面，在Github上已获得2.6K⭐️。
2. 支持的版本：

  - 完全支持： Android 14 - 16 <br>
  - 有限支持： Android 8.0 - 13
3. 主要特性如下
  - 丰富的安装类型：APK、APKS、APKM、XAPK、ZIP 内的 APK 和批量 APK。
  - 基于对话的安装
  - 基于通知的安装（支持实时活动 API）
  - 自动安装
  - 安装程序声明
  - 设置安装标志（可以继承配置文件设置）
  - 为特定用户/所有用户安装
  - Dex2oat 安装成功
  - 阻止安装特定应用的包名或共享UID
  - 安装后自动删除 APK 文件
  - 不使用 shell 命令，仅使用原生 API 调用。
  - 走 Android 原生通知，可适配部分系统灵动岛（比如ColorOS流体云）

## 三.简要使用步骤

1. 系统已经获取ROOT权限（ROOT之后体验最佳）。

- 只需要在设置里，将授权器更改为「ROOT超级用户」，再点击「锁定为默认安装器」，InstallerX-Revived就会接管后续的应用安装。

2. 无法获取ROOT权限，那就需要借用「Shizuku」或者「Dhizuku」等工具（因为系统默认安装器较高权限才能更改，且部分系统可能仍然会有限制，知悉）。

- 下载安装[Shizuku](https://www.123865.com/s/iMSmjv-3oLo)和[InstallerX-Revived](https://www.123865.com/s/iMSmjv-3oLo)。
- 默认状态下，Shizuku 处于未激活状态。考虑到旧设备，我们本期使用「电 脑ADB」的方式激活。
   - Android11及以上底层的设备可以考虑直接用WiFi调试激活，不需要电脑；
   - 点击Shizuku里面的“分步骤指南”跟着操作就行（不解BL锁也能玩机，Shizuku基础使用攻略：让应用使用adb高级权限）
3、进入手机系统设置-我的设备-全部参数和信息。连续点击5次「OS版本」，打开开发者选项（部分机型具有差异）

4、使用数据线链接电脑，在电脑上安装「」工具。
5、在设置-更多设置页面，找到开发者选项，开启USB调试、允许电脑进行调试
6、在电脑端搞机工具箱的「引导模式」选项卡中，找到「CMD命令行」，点击进入终端模式
7、将以下命令复制粘贴到终端，然后回车
```
adb shell /data/app/~~UwbNSwWUMQ1hInxN-FcBvA==/moe.shizuku.>privileged.api-5oBAvUDkWnm2Le7D60z4Xw==/lib/arm64/libshizuku.so
```
8、回到手机端，点击Shizuku的「激活」按钮，等待提示（Shizuku成功激活）
9、Shizuku成功激活后，点击「已授权应用」，选择给InstallerX-Revived授权。
10、授权成功后，在InstallerX-Revived点击「锁定为默认安装器」就可使用其替代安装应用了。
